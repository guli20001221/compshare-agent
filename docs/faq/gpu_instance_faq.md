## 1. 实例相关

### 1.1 实例卡初始化或卡启动中
容器启动失败，或者容器环境在使用中被破坏，会导致实例卡初始化或卡启动中，处理请加官方群联系群主。

### 1.2 实例常见错误码
- 错误代码 8357：资源不足，当前资源售罄，可以多刷新几次或者稍后购买。
- 错误代码 8095：配额不足，加官方群联系运营同学增加配额。
- 错误代码 8429：云主机资源已过期，请前往订单管理中手动支付欠费清单即可恢复。

### 1.3 容器实例如何设置服务自启动
可以在 `/start.d/` 目录下创建一个可执行的 bash 脚本，容器开机自动运行。

### 1.4 Windows 环境检测不到 GPU
由于不同配置、卡型驱动不同，Windows 环境需要用户在购买后自行安装显卡驱动。驱动下载地址：https://www.nvidia.cn/geforce/drivers/

### 1.5 Windows 镜像如何远程登录操作
Windows 镜像推荐使用本地电脑自带的远程桌面连接工具（mstsc），使用方便不卡顿。

### 1.6 Windows 镜像如何远程播放声音
需要在远程桌面连接设置中启用声音远程播放功能。

### 1.7 基础镜像如何查看 GPU、CPU、内存、磁盘信息
- GPU：输入 `nvidia-smi` 查看显卡信息。
- CPU 和内存：输入 `top` 命令查看。
- 磁盘信息：输入 `df -h` 查看。

### 1.8 SSH 无法连接、XShell 无法连接、VS Code 无法连接
请检查用户名、密码、端口号输入是否正确，不同类型镜像登录信息不同。

### 1.9 IP 是公网 IP 吗？
IP 是公网、固定 IP，可以当正常服务器对外提供服务使用。网络带宽为 1.5Gb 共享带宽，单 IP 上限 300Mbps。

### 1.10 关于"无卡启动模式"的使用和限制
- 无卡模式：需要在实例关机后，选择无卡模式启动（无卡模式不挂载 GPU，仅收取基础实例费 0.15 元/小时），适用于编写/调试代码、上传下载数据到实例的场景，节省资源成本。
- 注意事项：
  - 同一账号下仅允许 1 台实例开启无卡启动模式。
  - 目前无卡模式支持 4090、4090-48G、3090、5090、A800、H20。
  - 无卡开机不能制作镜像。

### 1.11 JupyterLab 关闭页面后服务器还会保持运算吗？
会继续保持运算。

### 1.12 Jupyter 中的 checkpoints 文件夹无法打开
checkpoints 是 JupyterLab 的关键字，跟 ComfyUI 冲突了，导致无法打开文件夹。解决方法：
1. 在 JupyterLab 的 Terminal 里执行 `cd checkpoints`，然后 `wget` 远程下载模型。
2. 使用第三方文件管理工具（如 XFTP）。

### 1.13 SD-WebUI、ComfyUI 服务经常断链
断链基本为显存、内存跑满导致服务崩溃，可以在控制台重启实例尝试。

### 1.14 nvidia-smi 检测不到显卡
可能为驱动问题，控制台重启实例尝试。如重启后仍然检测不到卡，则可判断为硬件故障，加群联系技术支持处理。

### 1.15 网页 Jupyter 左侧文件列表不显示
Jupyter 默认工作目录在 `/workspace` 下，即 `/workspace` 目录下的文件才会展示。请 `cd` 到 `/workspace` 目录下操作。

## 2. 计费相关

### 2.1 实例计费模式说明
- **按量**：按小时计费的后付费模式。关机后 CPU、GPU 和内存会被释放，实例不再收费；额外扩容的系统盘和数据盘及镜像资源会保留并继续收费。
- **包时**：按小时计费的预付费模式，已开通自动续费。关机后实例资源不会释放，仍然计费。
- **包日**：按天计费的预付费模式，已开通自动续费。关机后实例资源不会释放，仍然计费。
- **包月**：按月计费的预付费模式，已开通自动续费。关机后实例资源不会释放，仍然计费。

### 2.2 关于"卡初始化和启动中"的实例计费
- 实例卡初始化超过 5 分钟，以及卡初始化时间过长导致的扣费问题可进群联系群主解决。
- 实例卡启动中并不会产生扣费（启动中状态说明主机还并未开机）。

### 2.3 按量机器变更为包月
- 支持，根据客户自身需求，向客户经理/官方客户申请，通过后可非标支持。
- 变更包月的实例不支持再次变更计费方式，需要用户通过自制镜像后重新选择机器和计费重开。

### 2.4 账号存在欠费订单无法使用
每个账号下若存在 ≥1 条欠费订单则无法使用和创建新资源，请先前往"财务中心"手动支付后方可使用。

## 3. 镜像相关

### 3.1 镜像无法按照教程正常启动或使用
请加群联系群主，部分镜像可对接至原项目作者使用教学群。

### 3.2 自制镜像容量限制
当前自制镜像最大容量为 1000GB。

### 3.3 自制镜像大小
- 容器镜像（基于基础镜像制作）：镜像大小依据实际存储大小决定。
- 虚机镜像（基于系统镜像制作）：镜像大小依据系统盘大小决定。

### 3.4 什么类型的镜像不支持发布到社区
1. 使用系统镜像部署的实例（底层为虚机类型），无法发布至社区。
2. 使用社区镜像部署的实例，再次制作成镜像，不支持发布至社区。

### 3.5 关于制作镜像时的实例状态
- 基础镜像实例：需要保证开机状态下制作镜像。
- 系统镜像实例：与实例开关机状态无关，均可制作镜像。

### 3.6 关于"私有镜像"分享功能
使用场景：自制镜像一对一和一对多分享，支持分享者随时取消共享。
操作步骤：控制台 → 镜像列表 → 选择"平台镜像" → 更多"共享镜像" → 新建共享 → 输入被分享的用户 ID。
注意：这里的"私有镜像"指共享能力，不表示在"部署GPU实例"页面可直接选择私有镜像。

## 4. 实践相关

### 4.1 是否可以安装 Docker 和 Docker Compose
支持安装，自行安装需要使用系统镜像中的 Ubuntu 环境。安装命令：
```bash
sudo apt update
sudo apt install docker.io
sudo apt install docker-compose
```
也可以使用预装好的 Docker-Compose 环境的系统镜像。

### 4.2 Dify、CherryStudio 等工具如何调用云端 Ollama
在机器中使用如下命令启动 Ollama 服务：
```bash
export OLLAMA_HOST=你的外网IP:11434
ollama serve
```
在 Dify 客户端 API 地址中填写 `http://你的外网IP:11434`。
